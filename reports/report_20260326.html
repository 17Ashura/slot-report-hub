基本設計方針
演出システムの最重要原則：演出と戦闘処理は完全に分離する。
ProductionDirectorが演出を全てawaitで完了させてから、BattleManagerが処理を実行する。演出がゲームロジックに影響しない設計により、演出を後から追加・変更・スキップが自由にできる。
ターンフローの全体像
1
LeverOn → 内部抽選 + 演出抽選を同時実行
SlotManager.cs / ProductionLottery.cs
SlotLottery.Draw() で役・出目を決定すると同時に、ProductionLottery.Resolve(lotteryResult) を呼び出して演出パターンを決定。結果を ProductionPlan にパッケージ化する。
2
リール回転・停止 → ReelLayer が演出実行
ReelLayer.cs
リールが停止するたびに ProductionPlan.reelProductions[i] を参照し、フラッシュ・振動・点灯などの演出を実行。3リール全て停止後に次フェーズへ。
3
役確定 → WinLayer がファンファーレ・カットイン
WinLayer.cs
役の種類に応じた演出パターン（WinProductionType）を再生。ビッグウィンは CinemaLayer へ委譲するフラグも立てる。
4
BattleLayer → 攻撃エフェクト・SE
BattleLayer.cs
攻撃演出のタイプ（弱点攻撃・クリティカル・通常）に応じた演出を再生。ダメージ数字のポップアップもここが管轄。
5
CinemaLayer → ボス戦・特殊バフのシネマティクス
CinemaLayer.cs
条件（ボス出現・特定バフ発動・初回クリアなど）を満たした時のみ実行。通常は Skip として即時完了。Timeline / Animator連携。
6
全演出完了 → BattleManager が処理実行
BattleManager.cs（既存）
演出完了を待ってからダメージ計算・バフ適用・ターン終了判定を実行。既存コードへの変更は最小限（Director呼び出しを1行追加するだけ）。
既存システムとの接続方針
既存 BattleManager / BuffManager / BattleEventChannel への変更はゼロを目標とする。
BattleManager の ExecuteTurn() 内に await director.PlayAsync(plan) を1行挿入するだけで接続できる設計にする。

ENUM 各 ProductionType 列挙体
各レイヤーの演出種別を定義。追加は enum への値追加だけ。
  • ReelProductionType: None / Flash / Vibrate / Rainbow
  • WinProductionType: None / Small / Big / Jackpot
  • BattleProductionType: Normal / Weak / Critical / SpecialBuff
  • CinemaProductionType: None / BossAppear / FirstClear / SpecialBuff
各レイヤーは IProductionLayer を実装し、PlayAsync(plan) を await するだけで演出完了を保証する。
CinemaLayerは条件不成立時に即時完了（0フレーム待ち）するため、通常ターンのパフォーマンスに影響しない。

4つの演出レイヤー詳細
ReelLayer
REEL_STOP_PRODUCTION
トリガー：各リール停止時（3回呼ばれる）
演出種別：None / Flash / Vibrate / Rainbow
抽選元：出目パターン × ボーナス期待度
実装：DOTween / Animator 呼び出し
完了条件：アニメーション終了
WinLayer
WIN_FANFARE_PRODUCTION
トリガー：全リール停止後・役確定時
演出種別：None / Small / Big / Jackpot
抽選元：役の種類（ハズレ/リプレイ/小役/ボーナス）
実装：UIアニメーション・SE・カットイン
完了条件：演出終了 or プレイヤー入力
BattleLayer
BATTLE_ATTACK_PRODUCTION
トリガー：役確定後・バトル処理直前
演出種別：Normal / Weak / Critical / SpecialBuff
抽選元：弱点判定結果 + バフ状態
実装：ParticleSystem・カメラShake・数字ポップ
完了条件：エフェクト終了
CinemaLayer
CINEMATIC_PRODUCTION
トリガー：特殊条件成立時のみ
演出種別：None / BossAppear / FirstClear / SpecialBuff
抽選元：ゲーム状態（ボスHP・フロア・バフ種）
実装：Timeline / 専用Animator
完了条件：Timeline終了
レイヤー実行順序とスキップ戦略
順序レイヤー実行条件スキップ時
1ReelLayer常時（リールごとに実行）演出Noneで即完了
2WinLayer役確定時（ハズレも None で実行）演出Noneで即完了
3BattleLayer攻撃が発生するターン演出Noneで即完了
4CinemaLayer特殊条件成立時のみ即座にUniTask.CompletedTask
演出抽選テーブル設計
抽選テーブルは ProductionSelector (ScriptableObject) に完全外出し。
プログラマーが触らず、デザイナー・プランナーがInspectorで調整可能にする。

ProductionSelector — WinTypeごとの抽選テーブル
役の種類ごとに異なる演出抽選テーブルを持つ。SO なので Asset 差し替えで全て変更可能。
// ProductionSelector.cs
[CreateAssetMenu]
public class ProductionSelector : ScriptableObject
{
    [SerializeField] private WinTypeProductionTable[] tables;
    // WinTypeProductionTable: WinType + ProductionWeight[]

    public ProductionType Draw(WinType win)
    {
        var table = tables.First(t => t.winType == win);
        return WeightedRandom(table.weights);
    }
}

ProductionDirector — コア実装
public class ProductionDirector : MonoBehaviour
{
    [SerializeField] private ReelLayer    reelLayer;
    [SerializeField] private WinLayer     winLayer;
    [SerializeField] private BattleLayer  battleLayer;
    [SerializeField] private CinemaLayer  cinemaLayer;

    // BattleManager から呼ぶ唯一の入口
    public async UniTask PlayAsync(ProductionPlan plan)
    {
        // ① リール停止演出（リール側からコールバックで呼ぶ場合はここが空）
        await reelLayer.PlayAsync(plan);

        // ② 役確定ファンファーレ
        await winLayer.PlayAsync(plan);

        // ③ 攻撃エフェクト
        await battleLayer.PlayAsync(plan);

        // ④ シネマティクス（条件不成立時は即完了）
        await cinemaLayer.PlayAsync(plan);
    }

    public void SkipAll()
    {
        reelLayer.Skip();
        winLayer.Skip();
        battleLayer.Skip();
        cinemaLayer.Skip();
    }
}
BattleManager への接続（最小変更）
// BattleManager.cs — ExecuteTurn() への追加は1行のみ
public async UniTask ExecuteTurn(LotteryResult lotteryResult)
{
    // 既存: 演出計画の生成
    ProductionPlan plan = productionLottery.Resolve(lotteryResult);

    // ★ 追加: 演出を全て await
    await productionDirector.PlayAsync(plan);

    // 既存: バトル処理（変更なし）
    ProcessBattle(lotteryResult);
    buffManager.OnTurnEnd();
    CheckBattleEnd();
}
CinemaLayer — 条件分岐の実装例
public class CinemaLayer : MonoBehaviour, IProductionLayer
{
    [SerializeField] private Animator bossAppearAnimator;
    private bool isSkipped;

    public async UniTask PlayAsync(ProductionPlan plan)
    {
        isSkipped = false;
        switch (plan.CinemaProduction)
        {
            case CinemaProductionType.None:
                return;  // 即完了
            case CinemaProductionType.BossAppear:
                await PlayBossAppear();
                break;
            case CinemaProductionType.SpecialBuff:
                await PlaySpecialBuff(plan.LotteryResult.activatedBuff);
                break;
        }
    }

    public void Skip() => isSkipped = true;

    private async UniTask PlayBossAppear()
    {
        bossAppearAnimator.SetTrigger("Play");
        await UniTask.WaitUntil(() =>
            isSkipped ||
            bossAppearAnimator.GetCurrentAnimatorStateInfo(0).IsName("End"));
    }
}

設計方針の確定事項
✓
演出抽選は LeverOn 時に一括実行する
リール停止ごとに都度抽選する案もあったが、一括生成の方が演出の一貫性（例：3リール全てFlashで揃える）を保証しやすい。ProductionPlan が1ターンの演出を完全に記述する。
✓
バトル処理は演出完了後に実行する
「演出と処理を並列実行してUIをリアルタイム更新する」案は演出とロジックの結合度を高めるため採用しない。演出→処理の一方向フローを維持する。
✓
UniTask を使用する（Coroutine は使わない）
async/await によりネストが浅くなり、CompleteTask による即時スキップが自然に書ける。プロジェクトに UniTask 未導入の場合は IEnumerator + yield return で代替可能だが UniTask を推奨。
✓
各レイヤーは MonoBehaviour として Inspector 注入する
ProductionDirector へ IProductionLayer の配列として注入。レイヤーの追加・削除・並び替えが Inspector だけで完結する。
✓
ProductionPlan は使い捨て（ScriptableObject ではなく class/record）
1ターンに1枚生成され、演出完了後は GC に任せる。永続化・保存の必要はない。
