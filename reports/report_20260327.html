設計の3大原則（このセッションで確立）
①
バトル処理と演出の完全分離
LeverWaitState でバトルを即時完了（結果を隠蔽）し、各停止タイミングで演出を「見せるだけ」にする。演出をスキップしても結果は変わらない。
②
BattleEventChannel を演出計画の情報源にする
ProductionLottery が BattleEventChannel を購読してバトル結果を収集。TurnManager / BattleManager への変更ゼロで演出システムが接続できる。
③
ProductionRuleSet（SO）でルールをコード外に出す
「出目×HP残量×ラッシュ状態」などの条件と演出の組み合わせを Inspector で設定可能に。LotteryResult 追加時もコード修正不要。
確立されたフロー（確定版）
LeverState──▶Lottery.Pick() → 子役抽選確定
LeverWaitState──▶StartReelsWithDelayAsync() [リール回転開始]
└─▶ExecuteBattleAndBuildPlan()── バトル即時完了 → ProductionPlan 生成
FirstStopState──▶await PlayTimingAsync(plan.OnFirstStop)
SecondStopState──▶await PlayTimingAsync(plan.OnSecondStop)
ThirdStopState──▶await PlayTimingAsync(plan.OnThirdStop)
EndState──▶Replay → LeverState / 通常 → IdleState（ルーティングのみ）
既存コードへの変更方針
変更最小限を徹底。BattleManager / TurnManager / BuffManager への変更はゼロ。SlotManager への追加は3フィールド＋2メソッドのみ。Lottery.cs は1行追加（SetProductionPlan の呼び出し）のみ。
ProductionLottery の動き（核心）
// ① バトル前に購読開始
eventChannel.OnEventRaised += Collect;

// ② TurnManager.StartBattle() 実行
//    → BattleManager.ExecuteAttack()
//        → eventChannel.Raise(DamageApplied)  ← Collect() に流れる
//        → eventChannel.Raise(UnitDefeated)   ← Collect() に流れる
battleAction();

// ③ 購読解除
eventChannel.OnEventRaised -= Collect;

// ④ 収集結果 × ProductionRuleSet → ProductionPlan 生成
return BuildPlan(outcome);

Replay 仕様
Replay の場合、MaxBet（コイン消費）とバトルをスキップして直接 LeverState へ遷移する。
通常フロー：IdleState → MaxBetState → LeverState
Replay フロー：EndState → LeverState（MaxBet スキップ）
並走設計の考え方
リール回転中（LeverWaitState）にバトル処理を同期・即時完了させ、結果をユーザーに隠蔽する。各ストップステートで演出を流すことで「スロットが止まるたびに何かが起きる」体験を作る。演出を全スキップしてもバトル結果は変わらない。
