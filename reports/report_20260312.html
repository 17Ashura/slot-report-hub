仕様確定 確率バフはプレイヤー専用 BuffManager.cs
NotifyProbabilityChanged() 内の if (status is PlayerStatusRuntime) ダウンキャストは意図的。敵への確率バフは設計上存在しない。 
private void NotifyProbabilityChanged()
{
    // 確率バフはプレイヤー専用の仕様。敵への適用は想定していない。
    if (status is PlayerStatusRuntime player)
        player.NotifyProbabilityChanged();
}
仕様確定 CheckHit の weight は「命中率」 BattleCalculator.cs
missProbability という名前だが weight は命中率（0.8 = 80% で命中）。rnd < weight で命中判定するロジックは正しい。変数名 hitChance に修正済み。
仕様確定 ITurnOrderStrategy は UnitControllerBase 型を維持 ITurnOrderStrategy.cs
GetAllUnits() が List<UnitControllerBase> を返し、u.Status.speed の参照が必要なため IUnitController 型への変更はしない。MonoBehaviour に依存しないバトラーが必要になった時点で再検討。
仕様確定 OnTurnEnd の呼び出しはプレイヤーのみ slotManager 経由 PlayerStatusView.cs
プレイヤーは slotManager.OnEnd += Buffs.OnTurnEnd で自動的に呼ばれる設計。敵側は別の IPhase 実装から呼ばれる想定。意図的な非対称設計。
